# Knocking Tool — ابزار ناکینگ

برنامه ویندوزی برای اجرای **ناکینگ (Port Knocking) چندمرحله‌ای** با پروتکل‌های **ICMP**، **TCP** و **UDP**.

## راهنمای استفاده

### ۱. مدیریت نودها

هر **نود** یک مقصد شبکه است:

| فیلد | توضیح |
|------|--------|
| **نام نود** | نام دلخواه (مثلاً «سرور اصلی») |
| **IP مقصد** | آدرس IP سرور مقصد |

- **+ نود جدید:** افزودن نود
- **حذف:** حذف نود انتخاب‌شده

نودها **به‌صورت خودکار** ذخیره می‌شوند و با باز کردن مجدد برنامه بارگذاری می‌شوند.

**مسیر ذخیره خودکار:**
```
%AppData%\KnockingTool\config.json
```

---

### ۲. تعریف مراحل ناکینگ

برای هر نود می‌توانید یک **توالی چندمرحله‌ای** تعریف کنید:

| ستون | ICMP | TCP / UDP |
|------|------|-----------|
| **Order** | ترتیب اجرای مرحله | همان |
| **Protocol** | Icmp / Tcp / Udp | همان |
| **Port** | — (کاربرد ندارد) | پورت مقصد |
| **Size** | اندازه payload (بایت) | اندازه payload (UDP) |
| **Hdr 28B** | هدر IP+ICMP با raw socket | — |
| **Delay** | تأخیر تا مرحله بعد (ms) | همان |

#### ICMP
- **Size:** اندازه داده (payload) بسته
- **Hdr 28B:** اگر فعال باشد، بسته با هدر IP (20 بایت) + ICMP (8 بایت) ارسال می‌شود

#### TCP
- یک **SYN** به پورت مشخص ارسال می‌شود (برای port knocking کافی است)

#### UDP
- یک **datagram** به پورت و با اندازه مشخص ارسال می‌شود

---

### ۳. اجرای توالی

| دکمه | عملکرد |
|------|--------|
| **▶ اجرای نود** | اجرای توالی نود انتخاب‌شده |
| **▶▶ اجرای همه** | اجرای توالی همه نودها |
| **توقف** | متوقف کردن اجرای جاری |

---

### ۴. تکرار خودکار

| فیلد | توضیح |
|------|--------|
| **تعداد** | `1` = یک بار، `5` = پنج بار، **`0` = بی‌نهایت** (تا زدن توقف) |
| **فاصله (ms)** | زمان بین هر دور کامل اجرا |

---

### ۵. ذخیره / بارگذاری JSON

- **ذخیره JSON:** خروجی گرفتن از تنظیمات (پشتیبان یا انتقال)
- **بارگذاری JSON:** وارد کردن تنظیمات از فایل

---

### ۶. تم روشن / تیره

دکمه **🌙 / ☀️** در بالای صفحه — تم انتخاب‌شده خودکار ذخیره می‌شود.

---

### ۷. لاگ اجرا

پایین صفحه، نتیجه هر مرحله (موفق / ناموفق) با timestamp نمایش داده می‌شود.  
**پاک کردن** برای خالی کردن لاگ.

---

## مثال: Port Knocking سه‌مرحله‌ای

| ترتیب | پروتکل | پورت | تأخیر |
|-------|--------|------|-------|
| 1 | TCP | 1234 | 500 ms |
| 2 | UDP | 5678 | 500 ms |
| 3 | ICMP | — | 0 ms |
